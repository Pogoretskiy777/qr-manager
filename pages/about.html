QR Manager

  • About

Purpose

QR codes have been one of the most recognizable barcodes in the world since 1994. People typically see them everywhere in society for a variety of purposes, whether that be to see a restaurant’s menu or to join a club at a local university all from a single scan from one’s smartphone. Anyone can create a QR code and internet access has made it easier than ever. However, there has been a considerable lack of applications that allow users to save previously generated QR codes and organize them, whether that be for personal use or a company.

This website’s purpose is to enable users to generate and save these QR codes in an organized way for continuous maintenance. The problem this application aims to solve is QR code mismanagement and the lack of applications that allow users to organize and maintain their QR codes. This site also aims to simplify the updating of QR codes, as users can update their QR codes all in one place.

Additionally, the platform will offer features such as categorization, tagging, and search functionality, making it easier for users to locate specific QR codes quickly. Users will also have the ability to export or share their QR codes directly from the site.

Users

Any industry that uses QR codes in their organization can use this application. Below are listed a few general examples of users that can use this application:

Restaurant Owners

People who manage restaurants can use this application to generate QR codes for their menus. Oftentimes, QR codes are printed and put on dining tables for the guests to scan and either view the menu or pray for their order. This application makes it convenient to not only generate QR codes but also organize and keep track of different versions of the restaurant menus, whether that be based on their date of creation or the user-customized tags added alongside the QR code.

Student Organizations/Clubs

Students who manage university clubs often resort to generating QR codes for event signups or joining the club’s group chats. This application makes it convenient for students to generate QR codes and organize them in one convenient place to not lose track of which QR codes are used. It is also convenient to download the QR code from the get-go in their desired format.

This application will save time in organizing QR codes and knowing about the external links of the QR codes without having to scan them. Organizational tools for already created QR codes such as tags will allow users to keep track of their already created QR codes and make updating them much more convenient and user-friendly.

Features

The primary feature is generating custom QR codes given the style of the preferred code and the link it leads to. The following features are fields that the user can either read or read/write for each QR code generated:

  • Title
  • QR Code format/algorithm
  • Date of Creation
  • External URL Link
  • Tags (#)
  • Description

Each QR code can also be downloaded within the browser as a .png file. The user can also perform CRUD operations:

  • Create - Generate a QR code
  • Read - Read the generated QR code
  • Update - Update the QR code’s read/write data
  • Delete - Delete a generated QR code

The user can sort their QR codes by title, date of creation, and date last modified in either ascending or descending order. The user can also filter their QR codes by tags.

There will be a separate section of the site for each function. For example, there will be a dedicated screen for the organization, tagging, and updating of QR codes. The creation of a QR code can be accessed by clicking a button in the navbar, or the “Create new” button in the organizer.

Data

Users will be able to create, read, update, and delete QR code objects that will contain a name, format, date of creation, date last modified, external URL link, tags, and description. These QR codes can also be organized, sorted, and filtered by the user. The sort type and currently selected filter will also be created for each user, and can be read each time the site is visited, updated when changing these options, and can be reset to default by the user.

An example outline of the data used is as follows: a user generates a QR code, and an object for it containing the QR code data is generated. The QR code is assigned a Date of creation and last modified, an initial title of “Untitled QR Code”, and the format is generated based on what type of data the code is storing. The user can then specify the rest of the information. This means that the user can also create a new tag to assign to the QR code. Additionally, the user’s list of QR codes is updated. If a new tag is created, it will appear as an option for sorting and filtering.

API

QR Manager uses the goQR.me API to generate QR codes and fetching them to display for the user. There is no limit on the amount of QR codes a user can generate a day; however, the API's owners "reserve the right to reject API requests." This includes requests that are considered abusive or inappriopriate (i.e. a DoS attack) per discretion of the owners.

The owners of goQR.me do not store or log QR code contents, but they only know that a request to generate an API happened and have access to the origins of the request (referrer and IP address). The QR code image / symbol is deleted ~30 seconds after delivery from the cache and is not permanently stored. To learn more, visit the API's privacy policy page.
